# Counter App

Веб-приложение для подсчета посещений, разработанное для демонстрации навыков работы с контейнеризацией и микросервисной архитектурой.

Приложение построено на базе Python (Flask), использует PostgreSQL для хранения данных и Nginx в качестве обратного прокси-сервера. Развертывание осуществляется через Docker Compose.

## Стек технологий

*   **Backend:** Python 3.9 (Flask)
*   **Database:** PostgreSQL 13
*   **Web Server:** Nginx (Reverse Proxy)
*   **Containerization:** Docker, Docker Compose

## Архитектура и Безопасность

Проект реализует трехуровневую архитектуру с разделением сетей для повышения безопасности:

1.  **Сетевая изоляция:**
    *   В проекте настроены две внутренние сети: `frontend` и `backend`.
    *   Контейнер `nginx` и `app` находятся в сети `frontend`.
    *   Контейнер `db` и `app` находятся в сети `backend`.
    *   База данных не имеет публичного доступа (порты не проброшены на хост), взаимодействие с ней возможно только через приложение внутри сети Docker.

2.  **Управление данными:**
    *   Для сохранения данных PostgreSQL используется Docker Volume `db_data`. Это обеспечивает сохранность счетчика просмотров при перезапуске или обновлении контейнеров.

3.  **Конфигурация:**
    *   Параметры подключения к базе данных передаются через переменные окружения (Environment Variables) в `docker-compose.yml`, что соответствует методологии 12-Factor App.

## Установка и запуск

Для запуска требуется установленный Docker и Docker Compose.

1.  Клонирование репозитория:
    ```bash
    git clone https://github.com/charnes-test/counter-test.git
    cd counter-test
    ```

2.  Сборка и запуск контейнеров:
    ```bash
    docker compose up -d --build
    ```

3.  Проверка работоспособности:
    Приложение будет доступно по адресу: `http://localhost`

## API

*   `GET /` — Возвращает текущее количество посещений.
*   `GET /reset` — Сбрасывает счетчик посещений в ноль.
